bees 2022-01-31 18:22
you can span multiple remotes
yep
The workflow I have used is this:
1. Maintain a single docs repo, and multiple projects that need to be documented. 2. The docs repo should have a antora-playbook.yml that links to all modules to be supported. 3. Make a docs folder in the project, encompassing one or more modules 4. Each module can contain multiple attachments, images, and pages 5. Each project requires an antora.yml that define each of these modules 6. Each module requires at least an index.adoc file, written in asciidoc format 7. Once the above is compiled and in master or main, you should modify the docs repo's antora-playbook.yml to point to the new module(s).
