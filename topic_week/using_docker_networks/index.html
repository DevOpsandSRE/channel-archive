Nous 2021-10-25 16:29
99% of the time you really dont need to worry about docker networks. By default every container ends up in the default network and is able to talk to everything else in the same network. Docker-compose creates its own network for each file you deploy and everything inside a docker compose file can talk to everything else deployed in the same compose file. Changing the network to the host type is basically opting out of docker networks completely and just using the hosts network namespace (so localhost is the host box and you can see all traffic on the host) Generally you should not need this unless you need to talk to something on the host (and even then - why is that not in a container?). In production you should probably not be using docker networks beyond the default one. If your requirements grow to needing more then the one you probably already want to be using some orchestrate like Kube which has is own networking stuff and does not use any docker networks at all.
