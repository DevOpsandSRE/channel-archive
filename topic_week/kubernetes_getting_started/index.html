Darkwind 2022-01-05 15:17
Nginx frontend from Vue.js Master and Slaves Django/Celery Backends Two Redis DBs without persistence. Additional docker CLI app Initially I was deploying with Gitlab CI / Terraform / Docker-compose / Ansible / Dockerfiles Due to.... a bit self made approach, the code was overbloated in terms of additional Dockerfiles unefficiently used to add stuff, where it would be better to insert via Volumes (k8s allowed me just to point paths to files in configmaps) Ansible code was a bit overbloated? Migrating to managed k8s removed it. I had at ansible level basic server configurations / firewall-iptables shenanigans. Managed k8s removed the need, k8s exposes to public access only services to which I provided LoadBalancer/NodePort services. Scripts/(At ansible) which were needed just to point different parts of infrastructure to each other, where which part has which IP address, became not needed too. k8s internal DNS thing works by default without any code Basically all Ansible code was deleted. Unnecessary dockerfiles were removed, I can use basic images like Nginx without changes in k8s comfortably And Terraform code was all cut down to raise just one cluster + several node pools (with setting necessary node labels) and setting up domain records. That's it. (edited)
