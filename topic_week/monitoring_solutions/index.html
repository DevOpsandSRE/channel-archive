Nous 2021-11-15 09:01
I wouldn't instead of. Netdata collect high fidelity metrics (over 500 every second) but for only 1 hour. It is great at debugging issues with a server but not good for long term metrics - which is what prometheus is for. You can configure netdata to downsample and push to prometheus though so can use them togeather. But I would always go with prom for production workloads first. (edited)

Nous 2021-11-15 11:24
And to add to what I previously said. I tend to think of netdata more like the desktop system monitors on steroids that can also be used on servers. The usecase I generally use it for is debugging things when 30s/60s metrics are not good enough and you need secondly data during some event (like load testing a system to find bottlenecks). It can give you warnings/alerts about when some resources or limits have been reached and generally show you exactly where your system is struggling. Never used it in prod systems though, only ever local/test systems (though it is very lightweight so could be used in prod, just never had the need to).
