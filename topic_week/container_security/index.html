Pro tip: the less your container contains the less there is for an attacker to exploit. Languages that can be staticly compiled help a lot with this as you can drop a binary in a container without anything else. Multi stage builds also help as you often need more tools to build something then are required in the final image. Otherwise https://github.com/GoogleContainerTools/distroless containers help a lot as they contain just enough to run a language but without a package manager or shell that can be exploited.
